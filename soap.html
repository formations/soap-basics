SOAP Web Services Workshop

Workshop goal
The goal of this workshop is to provide a set of WSDL and XSD files to model books borrowing from a public library. This will be achieved step-by-step.

Modeling a borrowing service

Book
In order to borrow a book, the most important entity is the Book itself. A Book can be described by the following information:
  1. ISBN
  2. Name
  3. Author
  4. Date of publication
  5. Description
  6. Publisher
  7. Format
      The final result will look like this:
Create a new XML-Schema file named book.xsd.
Add the XML-Schema namespace in the relevant top element tag. If you don't remember the namespace, check the W3C specification. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema">
</schema>
Choose and set your own schema namespace. Remember that it has to be a URI, so either a URN or URL. Usually, a namespace is relevant to your organization. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
</schema>
In this new namespace, create an element with the name book. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <element name="book">
  </element>
</schema>
For each attribute, consider:
  • its data type, e.g. string, integer, etc.
  • its cardinality; min and max
  • whether it's mandatory (or not)
  • and whether to model it as an element or an attribute of the book element
author and publisher should be set as string (for now) 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <element name="book">
    <all>
      <element name="author" type="string" minOccurs="0"/>
      <element name="publisher" type="string" minOccurs="0"/>
    </all>
  </element>
</schema>
name and description are pretty evident 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <element name="book">
    <all>
      <element name="author" type="string" minOccurs="0"/>
      <element name="publisher" type="string" minOccurs="0"/>
      <element name="name" type="string"/>
      <element name="description" type="string" minOccurs="0"/>
    </all>
  </element>
</schema>
Find the correct data type for publicationDate. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <element name="book">
    <all>
      <element name="author" type="string" minOccurs="0"/>
      <element name="publisher" type="string" minOccurs="0"/>
      <element name="name" type="string"/>
      <element name="description" type="string" minOccurs="0"/>
      <element name="publicationDate" type="date" minOccurs="0"/>
    </all>
  </element>
</schema>
isbn is an ISBN-13. Design its datatype accordingly. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <element name="book">
    <all>
      <element name="isbn">
        <simpleType>
          <restriction base="string">
            <pattern value="[0-9]{13}"/>
          </restriction>
        </simpleType>
      </element>
      <element name="author" type="string" minOccurs="0"/>
      <element name="publisher" type="string" minOccurs="0"/>
      <element name="name" type="string"/>
      <element name="description" type="string" minOccurs="0"/>
      <element name="publicationDate" type="date" minOccurs="0"/>
    </all>
  </element>
</schema>
Refactor to create a dedicated type and set is as the type of isbn . 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  xmlns:tns="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <element name="book">
    <all>
      <element name="isbn" type="tns:isbnType">
      <element name="author" type="string" minOccurs="0"/>
      <element name="publisher" type="string" minOccurs="0"/>
      <element name="name" type="string"/>
      <element name="description" type="string" minOccurs="0"/>
      <element name="publicationDate" type="date" minOccurs="0"/>
    </all>
  </element>
  <simpleType name="isbnType">
    <restriction base="string">
      <pattern value="[0-9]{13}"/>
    </restriction>
  </simpleType>
</schema>
formatType is an enumeration whose value can be either ebook, paperback or hardcover. Create the type and add a new element of this type to book. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  xmlns:tns="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <element name="book">
    <all>
      <element name="isbn" type="tns:isbnType">
      <element name="author" type="string" minOccurs="0"/>
      <element name="publisher" type="string" minOccurs="0"/>
      <element name="name" type="string"/>
      <element name="description" type="string" minOccurs="0"/>
      <element name="publicationDate" type="date" minOccurs="0"/>
      <element name="format" type="tns:formatType"/>
    </all>
  </element>
  <simpleType name="isbnType">
    <restriction base="string">
      <pattern value="[0-9]{13}"/>
    </restriction>
  </simpleType>
  <simpleType name="formatType">
    <restriction base="string">
      <enumeration value="ebook"/>
      <enumeration value="paperback"/>
      <enumeration value="hardcover"/>
    </restriction>
  </simpleType>
</schema>
Finally, refactor the existing XML-Schema from a book element defining an anonymous type to define a bookType type in its own right. The book element should use this new type. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:book:2015"
  xmlns:tns="urn:unige:cui:library:book:2015"
  elementFormDefault="qualified">
  <complexType name="bookType">
    <all>
      <element name="isbn" type="tns:isbnType">
      <element name="author" type="string" minOccurs="0"/>
      <element name="publisher" type="string" minOccurs="0"/>
      <element name="name" type="string"/>
      <element name="description" type="string" minOccurs="0"/>
      <element name="publicationDate" type="date" minOccurs="0"/>
      <element name="format" type="tns:formatType"/>
    </all>
  </complexType>
  <simpleType name="isbnType">
    <restriction base="string">
      <pattern value="[0-9]{13}"/>
    </restriction>
  </simpleType>
  <simpleType name="formatType">
    <restriction base="string">
      <enumeration value="ebook"/>
      <enumeration value="paperback"/>
      <enumeration value="hardcover"/>
    </restriction>
  </simpleType>
  <element name="book" type="tns:bookType"/>
</schema>
Create a new XML document book.xml that uses the newly created XML-Schema. Take any book as an example; if you don't have idea, check this one. 
<?xml version="1.0"?>
<book xmlns="urn:unige:cui:library:book:2015">
  <isbn>9780749717100</isbn>
  <name>Of Mice and Men</name>
  <author>John Steinbeck</author>
  <publicationDate>1993-09-01</publicationDate>
  <description><![CDATA[A controversial...]]></description>
  <format>paperback</format>
</book>
User
The next step is to model the User. A User has many attributes, but in the context of this exercise, we'll restrict ourselves to these ones:
  1. Identifier
  2. First Name
  3. Last Name
Create a new XML-Schema file named user.xsd.
Add the XML-Schema namespace in the relevant top element tag. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema">
</schema>
Choose and set your own schema namespace, in accordance to the namespace of the previous schema. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:user:2015"
  elementFormDefault="qualified">
</schema>
In this new namespace, create a type with the name userType. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:user:2015"
  elementFormDefault="qualified">
  <complexType name="userType">
  </complexType>
</schema>
For each attribute, consider:
  • its data type, e.g. string, integer, etc.
  • its cardinality; min and max
  • whether it's mandatory (or not)
  • and whether to model it as an element or an attribute of the userType type
id should be unique 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:user:2015"
  elementFormDefault="qualified">
  <complexType name="userType">
    <all>
      <element name="id" type="ID"/>
    </all>
  </complexType>
</schema>
firstName and lastName are pretty self-evident 
<?xml version="1.0"?>

Library
The final entity to model is the Library itself.
Create a new XML-Schema file named library.xsd.
Add the XML-Schema namespace in the relevant top element tag. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema">
</schema>
Choose and set your own schema namespace, in accordance to the namespace of the two previous schemas. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:2015"
  elementFormDefault="qualified">
</schema>
In this new namespace, create a type with the name libraryType. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:2015"
  elementFormDefault="qualified">
  <complexType name="libraryType">
  </complexType>
</schema>
Add a new element with name id. It should be unique in the document. Don't forget to think about its cardinality. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  targetNamespace="urn:unige:cui:library:2015"
  elementFormDefault="qualified">
  <complexType name="libraryType">
    <all>
      <element name="id" type="ID"/>
    </all>
  </complexType>
</schema>
Add a new book element of type bookType to this new libraryType; remember to reference the correct namespace. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  xmlns:b="urn:unige:cui:library:book:2015"
  targetNamespace="urn:unige:cui:library:2015"
  elementFormDefault="qualified">
  <import schemaLocation="book.xsd"
    namespace="urn:unige:cui:library:book:2015"/>
  <complexType name="libraryType">
    <all>
      <element name="id" type="ID"/>
      <element name="book" type="b:bookType"/>
    </all>
  </complexType>
</schema>
Set the correct cardinality on the previous book element. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  xmlns:b="urn:unige:cui:library:book:2015"
  targetNamespace="urn:unige:cui:library:2015"
  elementFormDefault="qualified">
  <import schemaLocation="book.xsd"
    namespace="urn:unige:cui:library:book:2015"/>
  <complexType name="libraryType">
    <sequence>
      <element name="id" type="ID"/>
      <element name="book" type="b:bookType"
        minOccurs="0" maxOccurs="unbounded"/>
    </sequence>
  </complexType>
</schema>
Add a new user element of type userType to this new libraryType; remember to reference the correct namespace. 
<?xml version="1.0"?>
<schema xmlns="http://www.w3.org/2001/XMLSchema"
  xmlns:b="urn:unige:cui:library:book:2015"
  xmlns:u="urn:unige:cui:library:user:2015"
  targetNamespace="urn:unige:cui:library:2015"

Set the correct cardinality on the previous user element.

Service
Now it's time to model the web service itself. The service will contain a single operation to borrow a list of books.
Create a new WSDL file named library-service.wsdl.
Add the WSDL namespace in the relevant top element tag. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/">
</definitions>
Let's begin by the request message. It must contain the following:
  1. The library to borrow from of type libraryType
  2. The borrower of type userType
  3. The book to borrow
Create such a message and name it borrow_request. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/">
  <message name="borrow_request">
  </message>
</definitions>
Add all the attributes described above. Define the required type for each attribute. Don't forget about the namespace. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:l="urn:unige:cui:library:2015"
    xmlns:u="urn:unige:cui:library:user:2015"
    xmlns:b="urn:unige:cui:library:book:2015">
  <message name="borrow_request">
    <part name="library" type="l:libraryType" />
    <part name="user" type="u:userType" />
    <part name="book" type="b:bookType" />
  </message>
</definitions>
Now create the response message. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:l="urn:unige:cui:library:2015"
    xmlns:u="urn:unige:cui:library:user:2015"
    xmlns:b="urn:unige:cui:library:book:2015">
  <message name="borrow_request">
    <part name="library" type="l:libraryType"/>
    <part name="user" type="u:userType"/>
    <part name="book" type="b:bookType"/>
  </message>
  <message name="borrow_response">
  </message>
</definitions>
In a real-world service, the response would need to be descriptive, especially in the case of a failure. For now, just return a boolean. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:l="urn:unige:cui:library:2015"
    xmlns:u="urn:unige:cui:library:user:2015"
    xmlns:b="urn:unige:cui:library:book:2015">
  <message name="borrow_request">
    <part name="library" type="l:libraryType"/>
    <part name="user" type="u:userType"/>
    <part name="book" type="b:bookType"/>
  </message>
  <message name="borrow_response">
    <part name="return" type="boolean"/>
  </message>
</definitions>
Now design the portType and name it LibraryPort. Remember that port types are logical groupings. In this case, it's composed of a single operation. Name it borrow. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:l="urn:unige:cui:library:2015"
    xmlns:u="urn:unige:cui:library:user:2015"
    xmlns:b="urn:unige:cui:library:book:2015">
  <message name="borrow_request">
    <part name="library" type="l:libraryType"/>
    <part name="user" type="u:userType"/>
    <part name="book" type="b:bookType"/>
  </message>
  <message name="borrow_response">
    <part name="return" type="boolean"/>
  </message>
  <portType name="LibraryPort">
    <operation name="borrow">
    </operation>
  </portType>
</definitions>
An operation has 3 optional elements:
  • An input
  • An output
  • A fault
Set the input to the previously designed request and the output to the previous response. Leave the fault out for this workshop. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:l="urn:unige:cui:library:2015"
    xmlns:u="urn:unige:cui:library:user:2015"
    xmlns:b="urn:unige:cui:library:book:2015">
  <message name="borrow_request">
    <part name="library" type="l:libraryType"/>
    <part name="user" type="u:userType"/>
    <part name="book" type="b:bookType"/>
  </message>
  <message name="borrow_response">
    <part name="return" type="boolean"/>
  </message>
  <portType name="LibraryPort">
    <operation name="borrow">
      <input message="borrow_request"/>
      <output message="borrow_response"/>
    </operation>
  </portType>
</definitions>
The final step is to bind each WSDL element to its SOAP counterpart. Use the document literal style. 
<?xml version="1.0"?>
<definitions xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:l="urn:unige:cui:library:2015"
    xmlns:u="urn:unige:cui:library:user:2015"
    xmlns:b="urn:unige:cui:library:book:2015">
  <message name="borrow_request">
    <part name="library" type="l:libraryType"/>
    <part name="user" type="u:userType"/>
    <part name="book" type="b:bookType"/>
  </message>
  <message name="borrow_response">
    <part name="return" type="boolean"/>
  </message>
  <portType name="LibraryPort">
    <operation name="borrow">
      <input message="borrow_request"/>
      <output message="borrow_response"/>
    </operation>
  </portType>
  <binding name="LibraryBinding" type="tns:LibraryPort">
    <soap:binding style="document"
      transport="http://schemas.xmlsoap.org/soap/http"/>
    <operation name="borrow">
      <soap:operation soapAction="" />
      <input>
        <soap:body use="literal"/>
      </input>
      <output>
        <soap:body use="literal"/>
      </output>
    </operation>
  </binding>
  <service name="LibraryService">
    <port name="LibraryServicePort" binding="tns:LibraryBinding">
      <soap:address location="https://unige.ch/cui/library"/>
    </port>
  </service>
</definitions>
